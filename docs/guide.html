FEDIT 사용자 가이드

데이터로 분석하는 패션 트렌드의 새로운 기준 · 화면별 사용법 (일차안)

※ 목업 기준 초안 — 일부 수치·정의는 확정 후 보정 필요

0공통 화면 구성

0-1. 내비게이션

FEDIT의 기본 레이아웃은 좌측 세로 사이드바 + 상단 브랜드 바 구조입니다.

  • 좌측 세로 사이드바(고정): 실시간 아이템 · 상품 분석 · 색상 분석 · 유형 분석 · 패션쇼 분석
  • 상단 브랜드 바: 우측 브랜드 버튼으로 대상 브랜드를 선택하면, 현재 보고 있는 브랜드들이 상단에 가로 칩으로 나열됩니다. 이 브랜드 세트가 모든 분석 화면의 기준이 됩니다.
브랜드 122 ▾ADIDAS ✕NIKE ✕ CHANEL ✕브랜드 ✕…
위치요소설명
좌측 상단로고 / 사이드바 접기홈 이동 / 사이드바 축소
좌측 하단최근 본 항목최근 열람 상품 바로가기
좌측 최하단가이드 · 의견보내기 · 사용자도움말 · 피드백 · 계정(설정 진입)
우측 상단브랜드(예: 122) · 내 보드(저장함)브랜드 선택 모달 / 저장 보드
상단선택된 브랜드 가로 칩현재 분석 대상 브랜드 목록
상단(일부)어덜트 / 키즈 토글분석 대상 시장 전환

0-2. 브랜드 선택 (상단 브랜드 버튼)

  • 상단 우측 브랜드(선택 수 표시, 예: 122) 클릭 → 브랜드 필터 모달 오픈
  • 모달에 현재 선택된 브랜드가 함께 표시
  • 검색창 입력 또는 탭 탐색: 선택된 브랜드 · 즐겨찾기(준비중) · SPA 브랜드 · 명품 브랜드 · 디자이너
  • 브랜드 칩 선택/해제(00개 표시)
  • 해당 브랜드 데이터 보기 적용 / 선택 초기화하기 해제
  • 적용하면 선택 브랜드가 상단 가로 칩으로 뜨고 모든 분석이 그 세트 기준으로 갱신
💡
여기서 고른 브랜드가 상품·색상·유형·패션쇼 분석의 공통 기준 대상이 됩니다.

0-3. 공통 조작

  • 상단 어덜트/키즈 토글로 데이터 대상군 전환
  • 지표 옆 (?) 아이콘 호버 시 설명 툴팁
  • 👍 / 👎 는 AI 분석 결과 피드백(정확도 개선 반영)
  • 표/카드의 > 는 상세 펼치기, 데이터가 있는 화면은 Excel 내보내기 지원
🔒
지수·스코어 공통 원칙: FEDIT의 모든 지수(브랜드/상품/구매 화력도, 트렌드 스코어 등)는 대외비이며 플랫폼별·상품별 클릭·전환 등 여러 지표를 종합적으로 계산한 값입니다. 미입점 플랫폼의 데이터는 반영되지 않아 일부 값이 빈값으로 나올 수 있습니다.

1실시간 아이템 (홈 대시보드)

목적 지금 플랫폼에서 뜨는 검색어·트렌드 상품을 실시간 파악  ·  진입 좌측 메뉴 최상단 실시간 아이템

1-1. 플랫폼 내 인기 키워드 (상단)

무신사 · 29CM · W컨셉 3개 플랫폼의 인기검색어 TOP 10을 나란히 표시.

  • 날짜 선택 오늘(2025.01.16) ▾ : 기준일 변경  ·  이전달 / 다음달 이동
  • 순위 화살표: ▲ 상승 · ▼ 하락 · — 변동 없음
  • 세 플랫폼 교차 비교로 트렌드 범위(전체 vs 채널 특화) 판단

1-2. 트렌드 항목 (하단)

플랫폼 탭(무신사·29CM·W컨셉·통합) / 카테고리 탭(상의·아우터·바지·원피스/스커트) / 주차 이동.

① 트렌드 항목 리스트(좌) → 클릭 시 우측 패널 갱신
② 지표 카드(우상) — 상품이 "왜" 뜨는지 3축 분석

🔒
지수 산출 방식은 대외비입니다. 세 지수는 모두 플랫폼별·상품별 클릭·전환 등 여러 지표를 종합적으로 계산한 값이며, 구체 로직·가중치는 공개하지 않습니다. (아래 '참고 요소'는 해석용 예시일 뿐 산식이 아님)
⚠️
일부 플랫폼에 미입점된 상품/브랜드는 해당 플랫폼 데이터가 없어 지수가 빈값(또는 낮은 값)으로 표시될 수 있습니다. 값이 비어 보인다고 반드시 인기가 없는 것은 아니며, 입점 플랫폼 범위를 함께 고려해 해석하세요.
지표의미참고 요소(예시)
브랜드 지수브랜드 시장 반응 강도유사 브랜드 대비 시장 화력도
상품 지수상품 트렌드성·주목도유형 트렌드성, 찜하기 등
구매 화력도실제 구매 전환 강도판매수량, 리오더 여부(n차)

③ AI 개요 — 지표 종합 요약(예: '상업 검증 단계'), 상품 상세 보기 >로 이동
④ 디테일 유사 아이템 — 디자인 디테일이 비슷한 상품 추천(오버핏 등 태그)

읽는 순서  ▸  리스트(무엇) → 지표(왜) → AI 개요(어느 단계) → 유사 아이템(대응 상품군)

2상품 분석

목적 조건별 상품 탐색 + 개별 상품의 속성·지표·유사 상품 심층 분석  ·  진입 상품 분석

좌: 필터 → 중앙: 상품 그리드 → 우: 상품 상세 3단 구조.

  • 필터: 2025년·시즌 + 성별 · 유형 · 색상 · 디테일 · 소재 · 패턴. 선택값은 칩(자켓 ✕)으로 누적, 초기화로 전체 해제.
  • 그리드: 카드에 브랜드명 · 상품명 · 가격 · 누적조회수 · 누적판매 · 노출 플랫폼(무/W/29). 클릭 시 상세 오픈.
  • 상품 상세(예: DIOR – Caro 재킷)
    • 이미지, 정가/할인가, 성별, 신상 업데이트일, 노출 플랫폼 뱃지
    • 속성 분석: 색상·소매·기장·넥라인·핏·디테일·패턴 (각 👍/👎 피드백)
    • AI 개요 / 지표 카드(구매화력도·상품·브랜드 + 유형 트렌드성·실 품절 비율)
    • 유사한 스타일 아이템
    • 폴더명 ▾ → 저장하기로 내 보드에 저장
💡
속성 태그가 실제와 다르면 👎로 피드백 → 태그·추천 품질이 개선됩니다.

3색상 분석

목적 시즌 색상 트렌드 및 브랜드 색상 전략 비교, 신규 부상 색 포착  ·  진입 색상 분석

상단 시즌선택(예: 2-7), 우상단 그리드/리스트 보기 토글.

3-1. 색상비중 비교

  • 전체 브랜드 색상 분석: 시장 전체 색상 비중 트리맵(블랙 40% 등) + 전 시즌 대비 증감%
  • 브랜드별 카드(예: Dior): 특정 브랜드 색상 구성 비교
  • 브랜드 추가하기(+): 비교 대상 추가(카드 ✕로 제거)
  • 읽는 법: 블록 크기 = 비중, 뱃지 = 전 시즌 대비 변화

3-2. 신규 주목 색상 (트렌드 컬러)

"여러 경쟁사가 소량으로 동시 출시 중인 유행 색상"(예: 세이지 그린 88점).

  • 트렌드 스코어(예: 88점) · 성장 가속도 · 2개 이상 경쟁사 동시 출시 · 평균 비중/총 출시 수 · ⭐AI 추천
  • 하단: 브랜드별 비중(A.P.C. 4% 등) + 관련 아이템
🔒
트렌드 스코어 역시 대외비이며 여러 지표를 종합 계산한 값입니다. 입점 플랫폼 범위에 따라 일부 값이 비어 보일 수 있습니다.
💡
"소량 + 다수 브랜드 동시 출시 + 성장 가속" 조합 = 확산 직전 신호 → 초기 진입 색 후보.

4유형 분석

목적 최근 주목받는 상품 '유형'(카테고리·핏·소재)과 연관 키워드를 브랜드별 비교  ·  진입 유형 분석

전체 + 브랜드별(ADIDAS·NIKE·CHANEL·A.P.C.·Acne Studios·Jacquemus·The Kooples·Joseph 등) 카드를 격자로 나열합니다.

카드 구성

  • 헤더: 브랜드명 | 전체 합계(총 SKU)
  • 색상 비중 바: 상단 스택형 막대(카테고리 구성 비율). 막대 호버 시 값 툴팁(예: 아우터 49).
  • 카테고리 표: 카테고리 · 합계(SKU) · 비중 — 예: 상의 66(24%) · 스커트 63(23%) · 재킷 59(21%) · 트라우저 49(18%) · 반팔 티셔츠 42(15%)
  • 행 펼치기(>): 해당 카테고리의 세부 속성 순위를 핏 · 소재 · 기타 3열로 표시
    • 핏: 1위 루즈핏 · 2위 미디 기장 · 3위 맥시 기장
    • 소재: 1위 면 · 2위 폴리에스터 · 3위 레이온
    • 기타: 1위 프론트 슬릿 · 2위 드로우 스트링 · 3위 컷아웃 디자인
  • 우상단 Excel로 표 내보내기
💡
"전체" 카드로 시장 평균 유형 구성을 본 뒤, 특정 브랜드 카드와 대조해 어떤 카테고리·핏·소재에 치우쳐 있는지 확인합니다.

5패션쇼 분석 (Fashion Week Runway Analysis)

목적 10대 브랜드 런웨이를 비교해 시즌 트렌드 인사이트를 도출하고 실무 적용 포인트로 연결  ·  진입 패션쇼 분석

상단 Season Select(예: 2025 Spring/Summer)로 시즌 선택.

5-1. 시즌 전반 분석

  • 시즌 인사이트: 시즌 콘셉트 요약 문장(예: "과시 대신 여백, 구조보다 유연함…")
  • 대표 포인트 컬러: 컬러 스와치 + 명칭(피날레 레드·코발트 블루·헌터 그린·스톤 그레이 등)과 조합 코멘트
  • 주요 소재 & 텍스처: 핵심 소재 키워드(경량 트위드, 프린지/크러시드 가공 등) + 예시 썸네일
  • 실무 적용 포인트: 체크리스트형 제안(수트 셋업, 스커트 라인, 트위드 라인, 핏 조정 등)
  • 주요 아이템 & 디테일: 런웨이 이미지 그리드 + 라벨(크롭 트위드 재킷, 로우슬렁 스트레이트 팬츠, 오버사이즈 셔츠, 미디 스커트 등)

5-2. 브랜드별 컬렉션 분석

  • 브랜드 탭: Chanel · Miu Miu · Prada · Dior · Loewe · Bottega Veneta
  • 각 브랜드: 컬렉션 타이틀·시즌(예: 2025 S/S), 룩 설명, 자세히 보기
  • Color Palette: 브랜드 대표 색(아이보리·베이지·그레이·네이비 등)
  • 핵심 포인트: 룩별 요약(경량화 트위드로 봄 대응 / 미니 실루엣·로우라이즈 강조 / 시어 소재 믹스 / 클래식 디테일의 현대적 재해석 등)
💡
런웨이(5장)에서 잡은 컬러·소재·실루엣을 유형 분석(4장)·색상 분석(3장)의 실제 플랫폼 데이터와 대조하면 "쇼 트렌드 → 리테일 확산" 여부를 검증할 수 있습니다.

6내 보드 / 저장함

목적 관심 상품을 저장·비교하는 개인 작업 공간

  • 상품 분석 상세에서 폴더명 ▾ → 저장하기로 담은 아이템이 이곳에 모입니다.
  • 우측 상단 내 보드(저장함)에서 보드를 전환/열람.
  • 좌측 필터는 상품 분석과 동일하게 동작.
💡
기획 주제별 보드 분리(예: "25FW 아우터 후보")로 색상·유형·구매화력도를 한 번에 비교.

부록 B. 대표 사용 시나리오

  • 오늘 트렌드 빠르게 보기: 실시간 아이템 → 인기 키워드(큰 흐름) → 트렌드 항목(대표 상품·단계)
  • 상품 벤치마킹: 상품 분석 → 필터 → 상세(속성·지표·유사) → 내 보드 저장
  • 다음 시즌 색상 기획: 색상 분석 → 전체 vs 자사/경쟁 비중 → 신규 주목 색상 진입 후보
  • 브랜드 포지셔닝 진단: 브랜드 선택 → 유형 분석(카테고리/핏/소재 편중) → 색상 분석 대조
  • 쇼→리테일 검증: 패션쇼 분석 인사이트 → 유형/색상 분석의 실제 데이터로 확산 확인

📮 리뷰가 필요한 항목 (확정 요청)

  • 지수·트렌드 스코어는 대외비로 확정 → "종합 계산 값 + 미입점 시 빈값 가능" 수준까지만 노출 (반영 완료)
  • 유형 분석의 전체 합계(7000) 등 목업 수치 → 실데이터 표기 확정
  • 브랜드 필터의 즐겨찾기(준비중) 오픈 일정
